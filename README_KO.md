# 🛰 GitHub 인기 레이더

로컬에서 실행되는 잠재력 있는 GitHub 저장소 레이더로, 「X 핫스팟 레이더」와 대칭을 이룹니다 —— 다만 주시 대상이 GitHub일 뿐입니다.
「최근에 생성됨 + 이미 스타가 늘기 시작함」인 저장소를 포착해, 기술 트렌드, 콘텐츠 소재, 배울 만한 오픈소스 프로젝트를 발견하도록 도와줍니다.

## 무엇을 하나

- 4개 트랙을 스캔:**AI / LLM**, **Agent / MCP**, **크리에이터/콘텐츠 도구**, **해외 진출/인디 개발**
- 핵심 지표는 **스타 속도(⭐/일) = 총 스타 ÷ 생성 후 일수** 이며, 총 스타 수가 아닙니다 —— 일수가 적음 + 스타 속도가 높음 = 떠오르는 중
- 스냅샷을 기록하며, 두 번째 스캔부터 **🔥 지난번 대비 증가분**을 표시합니다(진정한 상승 신호)
- **실시간 급상승 랭킹(스캔 후 자동 추가)**: 스캔이 끝나면 스타 속도 상위 12개 후보에 대해 실제 star 타임스탬프를 자동으로 가져와 **현재 실시간 스타 속도**(최근 ~100개 star의 실제 속도)를 계산합니다. 「지금 막 터지는 중」인지 「이미 식었는지」를 구분할 수 있으며, 이는 평균 스타 속도로는 알 수 없습니다. 랭킹은 독립된 섹션으로 표시되고, 🟢실시간 태그는 트랙 목록의 같은 카드에도 표시됩니다. 참고: 4만 스타를 넘는 저장소는 GitHub의 페이지네이션이 상한에 도달하므로 근사값만 얻을 수 있습니다(UI에 🟢실시간 / ⚪근사 로 표시됩니다)
- 로컬에 로그인된 `gh` CLI 인증을 사용하므로 레이트 리밋이 높고 설정이 필요 없습니다
- 밝은 Apple 스타일 UI로, X 핫스팟 레이더와 일관됩니다

## 부팅 시 자동 실행 이미 설정됨(launchd)

이 서비스는 **부팅 시 자동 실행**으로 설치되어 있습니다(X 핫스팟 레이더와 동일한 launchd 방식이며, Homebrew의 node로 실행되어
데스크톱 디렉터리를 읽을 수 있습니다). 부팅 후 백그라운드에서 자동 실행되며, 언제든 아래 주소를 열기만 하면 됩니다:

**대시보드 주소 → http://127.0.0.1:8788**

### 더블클릭 스크립트

| 스크립트 | 역할 |
|------|------|
| `打开面板.command` | 웹 대시보드를 엽니다(실행 중이 아니면 자동으로 띄웁니다) |
| `重装自启服务.command` | 폴더를 옮겼거나 / 서비스가 비정상일 때, 재설치하고 시작합니다 |
| `卸载自启服务.command` | 중지하고 부팅 시 자동 실행을 해제합니다 |

> ⚠️ 폴더 전체를 **다른 곳으로 옮기면** 자동 실행이 작동하지 않습니다(plist에 옛 경로가 기록되어 있기 때문입니다).
> 옮긴 뒤 `重装自启服务.command` 을 한 번 더블클릭하면 고쳐집니다.

## 명령줄 전용 사용법(웹을 열지 않음)

```bash
python3 radar_core.py                 # 전체 트랙, 최근 90일, 스타 50개 이상
python3 radar_core.py --days 30       # 최근 30일의 새싹만 보기
python3 radar_core.py --track AI      # 특정 트랙만 스캔
python3 radar_core.py --min-stars 200 --top 15
```

## 대시보드 보는 법

- **🚀 스타/일**: 스타 속도, 높을수록 더 가파르게 상승 —— **이걸 가장 봐야 합니다**
- **📅 생성 후 일수**: 저장소가 생성된 지 며칠 됐는지, 작을수록 「초기 단계」
- **🔥 +N**: 지난 스캔 이후 늘어난 스타(두 번째 스캔부터 표시)
- **🟢 실시간 N/일**: 스캔 후 자동 추가되는 현재 실시간 스타 속도(최근 ~100개 star로 계산). 「바로 지금」 얼마나 빨리 크는지는 이것을 보세요
- 상단의 「지난 스캔 대비·스타 증가분 최다」 = 전체 트랙 중 가장 가파르게 오르는 Top10

## 트랙을 바꾸고 싶다면

트랙 키워드는 두 곳에 있습니다(어느 진입점을 쓰느냐에 따라 한 곳만 바꿔도 충분합니다):
- 웹 / 자동 실행 서비스:`server.mjs` 상단의 `TRACKS`
- 명령줄:`radar_core.py` 상단의 `TRACKS`

`topic:xxx` 는 GitHub 토픽 태그입니다.

## 파일 설명

| 파일 | 역할 |
|------|------|
| `server.mjs` | 로컬 웹 서비스(Node, launchd 자동 실행에서 사용) |
| `radar_core.py` | 스캔 핵심 로직(명령줄용) |
| `index.html` | 웹 대시보드 |
| `com.martin.github-radar.plist` | launchd 자동 실행 설정 템플릿 |
| `打开面板.command` / `重装自启服务.command` / `卸载自启服务.command` | 더블클릭 스크립트 |
| `snapshot.json` | 자동 생성되며, 지난 스캔을 기록(「증가분」 계산에 사용) |

## 환경

- 로그인된 GitHub CLI(`gh auth status` 에서 로그인 상태가 보임)
- Homebrew node(`/opt/homebrew/bin/node`)— 자동 실행 서비스에서 사용
- Python 3(명령줄용, macOS 기본 포함)
