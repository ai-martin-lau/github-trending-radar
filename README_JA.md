# 🛰 GitHub バズレーダー

ローカルで動く、有望な GitHub リポジトリ向けのレーダー。「X ホットスポットレーダー」と対をなすもので、見張る対象が GitHub というだけです。
「最近作成された + すでにスター数が伸び始めている」リポジトリを拾い上げ、技術トレンド、コンテンツのネタ、学ぶ価値のあるオープンソースプロジェクトの発見を手助けします。

## 何をするのか

- 4 つのトラックをスキャン:**AI / LLM**、**Agent / MCP**、**クリエイター/コンテンツツール**、**海外展開/個人開発**
- 中心となる指標は **スター速度(⭐/日)= 総スター数 ÷ 経過日数** であり、総スター数ではありません —— 経過日数が小さい + スター速度が高い = 急上昇中
- スナップショットを記録し、2 回目以降のスキャンから **🔥 前回からの増分** を表示します(本物の上昇シグナル)
- **リアルタイム急上昇ランキング(スキャン後に自動追加)**:スキャン完了後、スター速度上位 12 件の候補について実際の star のタイムスタンプを自動取得し、**現在のリアルタイムスター速度**(直近 ~100 個の star の実速度)を算出します。「今まさに伸びている」か「すでに冷めている」かを見分けられ、これは平均スター速度では分かりません。ランキングは独立したセクションとして表示され、🟢リアルタイム のタグはトラック一覧の同名カードにも付与されます。注:4 万スターを超えるリポジトリは GitHub のページネーションが上限に達するため、近似値しか取得できません(UI 上では 🟢リアルタイム / ⚪近似 と表示されます)
- ローカルでログイン済みの `gh` CLI 認証を利用するため、レート制限が高く、設定不要です
- ライトな Apple 風 UI で、X ホットスポットレーダーと統一されています

## 起動時自動実行をすでに設定済み(launchd)

このサービスは **起動時自動実行** としてインストールされています(X ホットスポットレーダーと同じ launchd 方式で、Homebrew の node で実行されるため、
デスクトップディレクトリを読み取れます)。起動後はバックグラウンドで自動実行され、いつでも下記のアドレスを開くだけです:

**ダッシュボード URL → http://127.0.0.1:8788**

### ダブルクリックスクリプト

| スクリプト | 役割 |
|------|------|
| `打开面板.command` | Web ダッシュボードを開く(動いていなければ自動的に起動します) |
| `重装自启服务.command` | フォルダを移動した / サービスが異常なときに、再インストールして起動します |
| `卸载自启服务.command` | 停止して起動時自動実行を解除します |

> ⚠️ フォルダ全体を **別の場所へ移動** すると、自動実行は無効になります(plist には旧パスが記録されているため)。
> 移動後は `重装自启服务.command` を一度ダブルクリックすれば直ります。

## コマンドラインのみの使い方(Web を開かない)

```bash
python3 radar_core.py                 # 全トラック、直近90日、スター50以上
python3 radar_core.py --days 30       # 直近30日の新しい芽だけを見る
python3 radar_core.py --track AI      # 特定のトラックだけをスキャン
python3 radar_core.py --min-stars 200 --top 15
```

## ダッシュボードの見方

- **🚀 スター/日**:スター速度。高いほど勢いが激しい —— **これを最も見るべき**
- **📅 経過日数**:リポジトリが作成されてから何日経ったか。小さいほど「アーリーステージ」
- **🔥 +N**:前回スキャン以降に増えたスター(2 回目以降のスキャンから表示)
- **🟢 リアルタイム N/日**:スキャン後に自動追加される現在のリアルタイムスター速度(直近 ~100 個の star から算出)。「今この瞬間」の伸びはこれを見る
- 上部の「前回スキャンから・スター増分が最多」= 全トラックの中で最も伸びている Top10

## トラックを変えたいとき

トラックのキーワードは 2 か所にあります(どの入口を使うかによって、片方を変えれば十分です):
- Web / 自動実行サービス:`server.mjs` 冒頭の `TRACKS`
- コマンドライン:`radar_core.py` 冒頭の `TRACKS`

`topic:xxx` は GitHub のトピックタグです。

## ファイル説明

| ファイル | 役割 |
|------|------|
| `server.mjs` | ローカル Web サービス(Node。launchd の自動実行で使用) |
| `radar_core.py` | スキャンの中核ロジック(コマンドライン用) |
| `index.html` | Web ダッシュボード |
| `com.martin.github-radar.plist` | launchd 自動実行の設定テンプレート |
| `打开面板.command` / `重装自启服务.command` / `卸载自启服务.command` | ダブルクリックスクリプト |
| `snapshot.json` | 自動生成され、前回のスキャンを記録(「増分」の算出に使用) |

## 環境

- ログイン済みの GitHub CLI(`gh auth status` でログイン済みが確認できる)
- Homebrew node(`/opt/homebrew/bin/node`)— 自動実行サービスで使用
- Python 3(コマンドライン用。macOS に標準搭載)
